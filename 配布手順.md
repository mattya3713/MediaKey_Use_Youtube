# Windowsメディアコントローラ 配布手順（C++常駐 + Chrome拡張）

## 1. 配布の基本方針
この構成は以下の2つをセットで配布します。

1. Windows常駐アプリ（`MediaKey.exe`）
2. Chrome拡張（`ChromeExtension` フォルダ）

---

## 2. 配布物の構成（推奨）
配布用フォルダ例:

- `MediaKey.exe`
- `zlib1.dll`
- `ChromeExtension/`  
  - `manifest.json`
  - `background.js`
  - `content.js`
- `INSTALL.md`（この手順を簡略化した利用者向け版）

> `MediaKey.exe` と同じ階層に `zlib1.dll` を置いてください。

---

## 3. 受取側インストール手順（ユーザー向け）

### 3.1 常駐アプリ配置
1. 任意フォルダに配布物を展開（例: `C:\Tools\MediaKeyController`）
2. `MediaKey.exe` と `zlib1.dll` が同じ場所にあることを確認

### 3.2 初回起動
1. `MediaKey.exe` を1回起動
2. バックグラウンド常駐・自動起動設定が反映される

### 3.3 Chrome拡張導入
1. Chromeで `chrome://extensions` を開く
2. 右上の「デベロッパーモード」をON
3. 「パッケージ化されていない拡張機能を読み込む」
4. `ChromeExtension` フォルダを選択

---

## 4. 動作確認
以下を押してYouTubeで動作確認:

- 次トラックキー → 10秒進む
- 前トラックキー → 10秒戻る
- 再生/一時停止キー → 再生切替

確認コマンド（cmd）:

```cmd
tasklist /FI "IMAGENAME eq MediaKey.exe"
netstat -ano | findstr :3001
```

期待結果:

- `MediaKey.exe` が表示される
- `127.0.0.1:3001` が `LISTENING`

---

## 5. よくあるトラブル

### 5.1 Chrome側 `ERR_CONNECTION_REFUSED`
原因: `MediaKey.exe` が起動していない。

対処:

```cmd
start "" "C:\Users\<ユーザー名>\AppData\Local\MediaKeyController\MediaKey.exe" --background
```

### 5.2 ウイルス対策ソフトで止まる
以下を例外登録:

- `MediaKey.exe`
- 配置フォルダ
- 必要なら `127.0.0.1:3001` ローカル通信

### 5.3 多重起動
最新版は単一起動制御あり。複数プロセスが残る場合は:

```cmd
taskkill /IM MediaKey.exe /F
```

---

## 6. 更新手順
1. 既存 `MediaKey.exe` を終了
2. 新しい `MediaKey.exe` / `zlib1.dll` に置換
3. `MediaKey.exe` を1回起動（再登録・再配置反映）
4. Chrome拡張も更新した場合は `chrome://extensions` で再読み込み

---

## 7. アンインストール手順
1. プロセス停止

```cmd
taskkill /IM MediaKey.exe /F
```

2. 配置フォルダ削除
3. 必要なら自動起動項目を削除

```cmd
reg delete HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v MediaKeyController /f
```

4. Chrome拡張を `chrome://extensions` から削除

---

## 8. 本番配布の推奨

- インストーラ化（Inno Setup / WiX）
- 実行ファイルのコード署名
- バージョン番号・更新履歴の管理
